#Surrogate + Inference Package for a Heat-Transfer Problem
Github Repo: https://github.com/AdnanEghtesad/Heat_Conduction_Surrogate.git

## The physics problem
We consider the steady-state heat conduction equation:
−k∇2T(x,y)=q
subject to Dirichlet boundary conditions:
T(x=0)=Tleft​,T(x=L)=Tright​
and geometry-dependent parameters:
p=[L,B,R,q,Tleft​,Tright​]∈R6
The objective is to learn the nonlinear operator:
G:p↦T(x,y)
where T(x,y) is the spatial temperature field defined over an irregular mesh.

#Geometry
A 2D rectangle, centred at the origin, with a circular hole in the middle:

```
        y = +breadth/2
   +-----------------------+
   |          ___          |
   |        /     \        |        x in [-length/2, +length/2]
   |       |  hole |       |        y in [-breadth/2, +breadth/2]
   |        \ ___ /        |        hole radius = radius, centred at (0,0)
   |                       |
   +-----------------------+
        y = -breadth/2
```

#Dataset
-The Dataset is generated by running "generate_dataset.py" with "num_samples" set to 100 in "dataset_config.yaml".
-Each sample consists of:
msh file: Mesh coordinates
vtp file: Solution field
json file: Design parameters 
-Note that each sample used a different geometry, undelrying mesh and boundary conditions

#Model
-The model uses a DeepONet Architecture
-The model is implemented in PyTorch
-The model can take the advantage of GPU
-GPU will be automaically selected if the machine/workstation has NVIDIA Graphics Card with the drivers installed 
-Live training visualization is implemented using Matplotlib.
-Evaluation metrics include:
Mean Absolute Error (MAE)
Root Mean Square Error (RMSE)
Min/max field comparisons
Uniformity

#Inference and Deployment
The trained model supports fast inference on unseen geometries. Given a new mesh and parameter set, the model outputs:	
T(x,y)≈G(p)

#How to use the Package
First setup the python envirrment and install the required libraries: 

## Setup and run 
-Install the libraries 
```
python3 -m venv .venv
.venv\Scripts\activate
pip install -r libraries.txt
```
-Run "Heat_Conduction_Surrogate.py"

#Output
-Predicted temperature fields are exported as .vtp files and written in each sample folder ("prediction.vtp") 
-The .vtp files can be visualized in opensource visualization toolkit, Paraview (https://www.paraview.org/)
-The ground truth temperature field ("solution.vtp") and ("prediction.vtp") could be both loaded into paraview to compare
the full-field temperature, range, min/max, as well as uniformity
-In addition to .VTP, the "Output.msg" contains the comparison of True and Predicted temperatures [First ten], as well as 
MAE (Mean Absolute Error) and RMSE (Root Mean Square Error), and temperature Uniformities (|∇T|) with their corresponding errors.  
-The trained, inference-ready model is saved as "Heat_Conduction_Surrogate.pth".
